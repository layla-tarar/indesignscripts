Quick summary of what the project does:
- clean_docx.py: Python script that pre-processes Word monograph drafts for InDesign placement. Converts superscripts to {{text}} markers, footnote refs to {{fn:N}}, repeat references to plain {{N}} superscripts. Outputs _clean.docx, _footnotes.txt, and _newterms.txt (candidate new terms to review before processing each monograph).
- TableStyler.jsx: Resets table overrides, converts first row to header, applies Table_Span, auto-detects approval vs. simple tables and applies TStyle_Approvals/TStyle_Simple with full cell styling. Run first.
- CleanUp.jsx: Remaps Word paragraph styles to InDesign styles, clears overrides, fixes text, restores superscripts, recovers Latin term italics. Run second (must be before InsertFootnotes).
- InsertFootnotes.jsx: Replaces {{fn:N}} markers with native InDesign footnotes from _footnotes.txt. Run third.
- TitleCaseHeadings.jsx: Applies APA 7th Title Case to all headings with term-specific fixes.
- FindDeleteEmptyFootnotes.jsx: Interactive cleanup of any unmatched {{fn:N}} markers.

The workflow is confirmed working for Cry1Ab through Phase 2.

Two-column layout (Phase 3 Step 1) is done manually: Cmd+A per spread → Cmd+B → 2 columns, 0.1667" gutter. Script automation for this was attempted and abandoned — InDesign crashes during reflow regardless of approach.

CleanUp.jsx includes Latin term italic recovery (step 6): applies Char_Italic / Char_Regular / Char_SemiboldItalic / Char_BoldItalic to terms like in vitro, in vivo, in situ, de novo, ex vivo, ad libitum etc., scoped per paragraph style weight. Journal name italics still require manual recovery.
