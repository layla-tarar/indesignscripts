Please read HANDOFF_NOTES.md and README.md before we start, so you have the full context.

Quick summary of what the project does:
- clean_docx.py: Python script that pre-processes Word monograph drafts for InDesign placement. Converts superscripts to {{text}} markers, footnote refs to {{fn:N}}, repeat references (empty-body Word footnotes with customMarkFollows) to plain {{N}} superscripts. Outputs _clean.docx and _footnotes.txt.
- TableStyler.jsx, CleanUp.jsx, InsertFootnotes.jsx, FindDeleteEmptyFootnotes.jsx, TitleCaseHeadings.jsx: InDesign ExtendScript scripts run in that order after placing _clean.docx.

The workflow is confirmed working for Cry1Ab through Phase 2.

Two-column layout (Phase 3 Step 1) is done manually: Cmd+A per spread → Cmd+B → 2 columns, 0.1667" gutter. Script automation for this was attempted and abandoned — InDesign crashes during reflow regardless of approach.

CleanUp.jsx now includes Latin term italic recovery (step 6): applies Char_Italic / Char_Regular / Char_SemiboldItalic / Char_BoldItalic to terms like in vitro, in vivo, in situ, de novo, ex vivo, ad libitum etc., scoped per paragraph style weight. Journal name italics still require manual recovery.
